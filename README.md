# Движок «Паспорта жизни» — расчётный блок

Превращает данные формы в проверенный JSON для HTML-шаблона PDF.
**Все астрономические расчёты — программные** (Swiss Ephemeris через `kerykeion`).
ИИ задействован только для литературного текста и работает с уже верными фактами.

## Файлы

| Файл | Назначение |
|---|---|
| `chart_engine.py` | Ядро: данные формы → натальная карта (планеты, дома, аспекты). Расчёт детерминирован. |
| `forecast.py` | Транзиты на 2026–2028 (тоже детерминированы). |
| `interpret.py` | **Единственное место с ИИ.** Получает факты → пишет тексты. Есть офлайн-режим на шаблонах. |
| `build_passport.py` | Оркестратор: собирает финальный JSON в формате шаблона. |
| `api.py` | HTTP-эндпоинт `POST /api/passport` (FastAPI). |
| `requirements.txt` | Зависимости. |

## Конвейер

```
форма → ChartInput → compute_chart() ──┐
                     transits_for_year()├─→ build_passport() → JSON → HTML-шаблон → PDF
                     interpret(llm=True)┘        (факты от движка,
                       ↑ ИИ пишет текст            тексты от ИИ)
                         по фактам
```

## Установка

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-...   # только если use_ai=True
```

## Запуск API

```bash
uvicorn api:app --reload
```

Запрос (поля = поля вашей формы):

```bash
curl -X POST http://localhost:8000/api/passport -H "Content-Type: application/json" -d '{
  "name": "Александр",
  "dob": "13.06.1990",
  "tob": "12:30",
  "pob": "Гомель, Беларусь",
  "time_unknown": false,
  "events": [
    {"date":"20.07.2016","label":"Свадьба"},
    {"date":"05.03.2019","label":"Рождение ребёнка"}
  ],
  "use_ai": true
}'
```

Ответ — JSON с ключами `meta, chart, key_aspects, life_mission, strengths,
challenges, life_periods, key_events, yearly_forecast_next3`. Его подставляете
в `const DATA = {...}` HTML-шаблона.

## Важные детали

- **Часовой пояс и DST** определяются автоматически из координат
  (`timezonefinder`) + исторические переходы (`pytz`). Ручной ввод смещения
  не нужен — это убирает класс ошибок с летним/зимним временем.
- **Геокодинг**: если на фронте уже есть координаты места — передавайте
  `lat`/`lng`, тогда расчёт идёт офлайн без обращения к Nominatim.
- **Время неизвестно** (`time_unknown=true`): карта строится на 12:00,
  дома помечаются условными (`meta.time_known=false`). Ректификацию по 5
  событиям можно добавить отдельным модулем (следующий этап).
- **`use_ai=false`**: тексты собираются из шаблонов `interpret.TEMPLATES` —
  стабильно, без сети. Удобно для тестов и как fallback при недоступности ИИ.

## Где расширять

- Тексты-шаблоны и системный промпт ИИ — в `interpret.py`.
- Орбисы аспектов и набор отображаемых планет — в `chart_engine.py`.
- Глубина транзитов (орб, набор планет) — в `forecast.py`.
