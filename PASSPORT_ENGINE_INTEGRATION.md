## 2. Контракт (вход → выход)

**Вход** — то, что собирает форма. Маппинг полей формы → поля движка:

| Поле формы | Поле движка | Обязательное |
|---|---|---|
| Имя | `name` | да |
| Email | `email` | нет (для доставки PDF) |
| Дата рождения | `dob` (`"ДД.ММ.ГГГГ"`) | да |
| Время рождения | `tob` (`"ЧЧ:ММ"`) | да, если не стоит галка |
| «Не знаю точное время» | `time_unknown` (bool) | — |
| Место рождения | `pob` (текст) | да |
| 5 событий | `events` (список `{date, label}`) | нет |

**Вызов из кода:**

```python
from chart_engine import ChartInput, LifeEvent
from build_passport import build_passport

inp = ChartInput(
    name="Александр",
    dob="13.06.1990",
    tob="12:30",
    time_unknown=False,
    pob="Гомель, Беларусь",
    events=[LifeEvent("20.07.2016", "Свадьба"),
            LifeEvent("05.03.2019", "Рождение ребёнка")],
)

data = build_passport(inp, use_ai=False)   # <-- СЕЙЧАС use_ai=False (см. раздел 4)
```

**Выход** — `data` это `dict` с ключами ровно под наш HTML-шаблон:
`meta, chart, key_aspects, life_mission, strengths, challenges,
life_periods, key_events, yearly_forecast_next3`.

## 3. Куда подставить результат

В `life_passport_template.html` есть строка вида `const DATA = { ... };`.
Нужно подставить туда сериализованный `data`. Способ зависит от того, как
у нас рендерится PDF — выбери подходящий и реализуй:

- **Серверный рендер (Puppeteer/WeasyPrint и т.п.):** прочитать шаблон как
  строку, заменить блок `const DATA = {...};` на `const DATA = <json>;`
  (json = `json.dumps(data, ensure_ascii=False)`), отдать в рендерер.
- **Отдельный JSON для фронта:** вернуть `data` как ответ API, а на странице
  результата подставить его в шаблон до генерации PDF.

Не меняй разметку и стили шаблона — только данные.

## 4. ⚠️ ВАЖНО: сейчас работаем БЕЗ ИИ

На текущем этапе движок **полностью детерминирован и не использует ИИ**:

- Все **расчёты** (планеты, дома, аспекты, транзиты, часовой пояс с DST) —
  это чистая астрономия (Swiss Ephemeris). ИИ там не было и не будет.
- **Тексты** (предназначение, сильные стороны, описания периодов и прогноза)
  сейчас собираются из встроенных шаблонов в `interpret.py`.
- Поэтому **`use_ai=False`** во всех вызовах `build_passport(...)`.
- **`ANTHROPIC_API_KEY` сейчас НЕ нужен**, библиотека `anthropic` для работы
  в этом режиме не требуется. Сетевых вызовов к ИИ нет.

Что это значит для интеграции: ставь дефолт `use_ai=False`, не добавляй
обязательную зависимость от ключа ИИ, не блокируй генерацию паспорта на ИИ.
Пайплайн должен работать офлайн на расчётах и шаблонах.

### ИИ подключим позже (заложи задел, но не включай)

Единственная точка подключения ИИ — функция `interpret(..., llm=True)` в
`interpret.py`. Когда дойдём до этого этапа:

1. добавим `anthropic` в зависимости и `ANTHROPIC_API_KEY` в окружение;
2. переключим вызов на `build_passport(inp, use_ai=True)`;
3. ИИ будет получать **уже посчитанные факты** и только превращать их в живой
   текст — он по-прежнему ничего не вычисляет и не выдумывает астрологию.

Сделай переключатель управляемым через конфиг/переменную окружения
(например `PASSPORT_USE_AI=false`), чтобы потом включить ИИ без правок кода.

## 5. Геокодинг и часовой пояс

- Часовой пояс и историческое летнее/зимнее время определяются **автоматически**
  из координат места рождения (`timezonefinder` + `pytz`). Руками смещение
  вводить не нужно.
- Если на фронте уже есть координаты места (автокомплит/карта) — прокидывай
  `lat`/`lng` в `ChartInput`, тогда расчёт идёт офлайн без обращения к Nominatim.
- Если координат нет — движок геокодит текст места через Nominatim (нужен
  интернет на сервере). Учти rate-limit Nominatim; для прода лучше свой геокодер
  или кэш.
